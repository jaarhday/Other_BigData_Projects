# Challenge General Information

You can read the details of the challenge at [challenge.md](challenge.md)

## Key Items

- __Due Date:__ 3/11
- __Work Rules:__ You write your own code building your unique features. You can talk with your team about the logic, but they can't write your code.
- __Product:__ Based on the description in the [challenge.md](challenge.md), you will deliver your work as the following products.
     1. Team Documentation: A Github pull request that provides a data dictionary for all your columns (features) and documentation on how each team member builds their feature.
     2. Grade Submission: The code and documentation for your specific part of the work. It can be your team documentation with your name clearly showing where your work was done. This is submitted in Canvas.
- __Databricks Process:__ Each team member will export their workbooks as `.ipynb`s and their data (first 100 rows) as a `.csv`.
- __Canvas Process:__ Each student will upload a `.pdf` or `.html` files with your results as described in [challenge.md](challenge.md)
- __Class Presentation Process:__ On the due date, each team will present plots of their features against their target with a discussion of usability.
